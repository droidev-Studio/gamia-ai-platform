## Required Prompt

```text
I want to generate a cozy animal-island inspired bullet-hell flying shooter. Keep the core gameplay as Bullet Hell / Flying Shooter: the player pilots a tiny leaf-wing glider with auto-fire, dodges readable projectile patterns, collects shields, fruit bombs, and energy blossoms, and progresses through 3 to 4 staged boss battles.

The visual style should be extremely different from cyber-neon: warm cream background, mint green, peach, soft leaf green, rounded toy-like silhouettes, handmade paper-cut shapes, soft shadows, pill-shaped UI, cozy village-island charm, gentle nature motifs, acorn buttons, flower bullets, fruit pickups, cloud puffs, and friendly animal-island atmosphere.

Do not use dark cyberpunk, neon grids, sci-fi metal, harsh glows, photorealism, or copyrighted Animal Crossing characters/logos. The result should feel like a cozy animal island arcade shooter, while still preserving bullet dodging, boss phases, bombs, shields, waves, and clear hitbox readability.
```

## Pass Criteria

- The test starts from the local frontend UI.
- The AI plan is confirmed from the local UI.
- The browser remains on the local frontend after generation.
- Generated project files are visible in the frontend Code slot.
- The game remains `bullet_hell`.
- The uploaded template runtime is used.
- Animal-Island visual runtime and CSS are present.
- Manifest contains the required art prompt phrases.
- Boss progression does not collapse to one boss.
- No `game.js` runtime crash is observed.

## Fail Criteria

- The current tab is redirected to `/generated/...` instead of staying in the local workspace.
- Code slot shows placeholder files only.
- Code slot has zero generated files.
- `templateId` changes away from `bullet_hell`.
- Compile falls back to builtin runtime unexpectedly.
- Boss count becomes 1.
- Manifest lacks `flower bullets`, `fruit pickups`, or `pill-shaped UI`.
- Console shows `Cannot read properties of undefined (reading 'length')` or another gameplay-breaking runtime error.
